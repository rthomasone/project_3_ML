Machine Learning Models

Recently released songs were processed by the model to predict their genre.

Modeling

We split the sample data into a test data(80%) and train data(20%), balanced our features by scaling them and finally, we used the scaled data for creating predictive models.

Supervised learning classification algorithms were used to predict which genre a track would belong too. The 3 models We used were: Logistic regression analysis, random forest classifier and Xtreme gradient boosting classifier.



Clustering

The original Spotify data set has over a hundred different genres/sub-genres. How can we know how many genres to narrow it down to? We looked to K-Means Clustering Analysis and Silhouette Analysis for some insights. The K-Means clustering algorithm suggested that between 4-6 genres would be most optimal. The team felt that 4-6 genres was too small. The silhouette analysis suggested that 4 or 12 genres would be most optimal. Twelve genres was chosen.

Logistic Regression

The first model attempted was the logistic regression because it is the simplest. It works by using the sigmoid function (logistic function) to map and predict the probabilities of a track falling into a genre. If the probability of a track were closer to 1, it will be transformed into a binary value of 1(meaning it falls into that category). Else, it will be transformed into a binary value of 0(meaning it does not fall into that genre).

Random Forest

Since we were tackling our models in the order of simplicity, the next model was the random forest classifer; an ensemble tree based learning algorithm that combines decision trees.

XGBoost

The final model that we tested was the extreme gradient boosting classifier. It is an implementation of gradient boosted decision trees designed for speed and performance



Comparison & Selection

We used the ROC curve, Accuracy, Precision and Recall metrics to decide on the best model for predicting genres.These were the results:
